Exercise 1: Filter üîç

Filter inventors born in the 1500s

Your Task:

Use filter() to find all inventors born between 1500-1599

  // What condition goes here?

Exercise 4: Reduce üìä

Calculate total years all inventors lived

Your Task:

Use reduce() to sum up all the years lived by inventors

const totalYears = inventors.reduce((total, inventor) => {
  // Calculate lifespan and add to total
  return ???;
}, 0);

// Hint: passed - year = lifespan

Solution: Reduce ‚úÖ

const totalYears = inventors.reduce((total, inventor) => {
  return total + (inventor.passed - inventor.year);
}, 0);

console.log(totalYears); // 861 years total!
Reduce pattern: accumulator + current calculation, initial value

The some() Method üîç

Is at least one person 19 or older?

const isAdult = people.some(person => {
  const currentYear = new Date().getFullYear();
  return currentYear - person.year >= 19;
});

console.log(isAdult); // true or false
some(): Returns true if ANY element passes the test

The every() Method ‚úÖ

Is everyone 19 or older?

const allAdults = people.every(person => {
  const currentYear = new Date().getFullYear();
  return currentYear - person.year >= 19;
});

console.log(allAdults); // true or false
every(): Returns true only if ALL elements pass the test

The find() Method üéØ

Find the comment with ID 823423

const comment = comments.find(comment =>
  comment.id === 823423
);

console.log(comment);
// { text: 'Super good', id: 823423 }
find(): Returns the FIRST element that matches (or undefined)

The findIndex() Method üìç

Find the index of comment 823423

const index = comments.findIndex(comment =>
  comment.id === 823423
);

console.log(index); // 1

// Use it to delete:
comments.splice(index, 1);
findIndex(): Returns the index position (or -1 if not found)

Method Comparison üìä

Transform Methods

map(): New array (same length)
filter(): New array (fewer items)
reduce(): Single value

Test Methods

some(): Boolean (any match?)
every(): Boolean (all match?)
find(): First match or undefined
findIndex(): Index or -1

Performance Tips ‚ö°

Write efficient array code

‚ùå Less Efficient

// Multiple passes

const names = adults.map(u => u.name);

‚úÖ More Efficient

The Transformation ‚ú®

Code Before and After ES5

Life Before ES5 (Pre-2009):

// Get even numbers and double them
var numbers = [1, 2, 3, 4, 5, 6];
var result = [];

for (var i = 0; i < numbers.length; i++) {
  if (numbers[i] % 2 === 0) {
    result.push(numbers[i] * 2);
  }
}
console.log(result); // [4, 8, 12]

Life After ES5 (2009+):

// Get even numbers and double them
const numbers = [1, 2, 3, 4, 5, 6];

const result = numbers
  .filter(n => n % 2 === 0)
  .map(n => n * 2);

console.log(result); // [4, 8, 12]

Array Method Best Practices ‚≠ê

Golden Rules:

1. Choose the right method for the job
2. Chain when it improves readability
3. Use descriptive variable names
4. Don't mutate unless you mean to (avoid sort)
5. Remember: map/filter/reduce return NEW arrays

When in doubt: Break it down into steps and test each one!
