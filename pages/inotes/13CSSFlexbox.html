Historical Context üìú

The Dark Ages of Table-Based Layouts

  • Twenty-plus years ago, tables were the main way web pages were created in HTML
  • It gave web builders consistent control of constructing pages with some "design"
  • No longer did sites only have to be top-to-bottom in a linear manner ‚Äî they could be set up with columns
  • Back then, it was seen as a huge breakthrough!
<table><tr><td>Layout</td></tr></table>

Tables, however, were never designed to lay out pages and have all sorts of problems when used that way today. It was a convenient hack, but at the time, a very welcome one.

Source: CSS-Tricks - "In Defense of Tables and Floats in Modern Day Development"

What is Flexbox?

Flexible Box Layout = A one-dimensional layout method

‚úÖ What Flexbox IS

  • A way to arrange items in rows OR columns
  • Items that grow/shrink based on available space
  • Perfect for component layouts
  • Great for navigation bars, cards, etc.

‚ùå What Flexbox ISN'T

  • Not for complex 2D layouts (use Grid)
  • Not a replacement for all positioning
  • Not always the best solution
  • Not supported in IE9 and below

‚ö†Ô∏è Common Flexbox Gotchas

Watch Out For These!

  • 1. Margin Collapse Doesn't Happen

    Vertical margins don't collapse in flex containers

  • 2. Min-Width: Auto

    Flex items won't shrink smaller than their content by default

  • 3. Default Alignment

    Items stretch to fill container height by default

  • 4. Percentage Heights

    Can be tricky in flex containers

Solution: Use DevTools to debug these issues!

Historical Context üèóÔ∏è

Float Layouts: Building Houses with Duct Tape and Broomsticks

  • With the push toward web standards in the early 2000s, web authors began to hack out layouts with complicated mixtures of absolute positioning and floats
  • CSS Layout at that time was mostly built out of this
  • As one expert described it: "It's an ingenious use of the materials available, but a bit like building a house out of duct tape and broomsticks."
  • Floats were not intended for coarse layout but for text wrapping around images
  • Tables were still the accepted way of handling 2D layout; there was nothing else
.column { float: left; width: 33.33%; }
.clearfix::after { clear: both; /* The horror! */ }

The complexity of float-based layouts led to countless debugging hours and the infamous "clearfix hack" that developers had to memorize.

Source: fantasai - "Evolution of CSS Layout: 1990s to the Future"
