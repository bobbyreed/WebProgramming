Slide 4: Breaking Down the Design

0:35-0:42 (7 minutes)

What to Say:

  • "Before you code, you need to analyze the design. What sections do you see?"
  • "Let's identify them together: Hero with nav, Information cards, Quote, Call-to-action, Footer."
  • "Each section is a container. Think about how you'll structure your HTML first."
Use your cursor or drawing tool to outline each section on the design image. Draw boxes around them.

Key Teaching Points:

  • Hero section spans full width with dark background
  • Information section has 4 cards - perfect for Flexbox!
  • Quote section uses italics and different background
  • CTA (Call-to-Action) section is similar to hero but smaller
  • Footer is simple, centered text
Ask students: "If you were building this, which section would you start with?" Most say hero - validate this approach.

Slide 17: Let's Build!

0:60-0:80 (20 minutes - Work Time)

What to Say:

  • "Alright, it's work time! Start with your Git repository setup if you haven't already."
  • "I'll be walking around. Raise your hand if you get stuck for more than 5 minutes."
  • "Remember: HTML first, CSS second. Structure before style."

During Work Time - Your Role:

  • Circulate constantly - Don't sit at your desk
  • Check in with quiet students - "How's it going? Need anything?"
  • Answer questions with questions - "What have you tried? What did you expect to happen?"
  • Debug together - Don't just give answers, guide them to solutions
  • Celebrate small wins - "Great! Your hero section looks good!"
If you see 3+ students stuck on the same issue, pause the class for a 2-minute mini-lesson addressing it.
Keep VS Code open with a working example. If a student is completely lost, show them your code briefly, then have them close your screen and recreate it.

Common Issues You'll See:

  1. Flexbox not working: "Did you set display: flex on the parent?"
  2. Colors not matching: "Use the exact hex codes from the design spec"
  3. Things not centering: "Align-items for vertical, justify-content for horizontal"
  4. Box model confusion: "Draw out the box model on paper first"
  5. Git issues: "Did you initialize the repo? Is your remote set up?"

COMMON STUDENT QUESTIONS & ANSWERS

  1. Q: "Do I have to use the exact same colors and fonts?"
    A: Colors yes (practice matching specs), fonts can be similar if the exact ones are hard to get.
  2. Q: "Can I change the content to be about something else?"
    A: Absolutely! In fact, I encourage it. Make it yours.
  3. Q: "My nav links don't go anywhere. Is that okay?"
    A: Yes, use # as the href. They're placeholders on a single-page site.
  4. Q: "Should I use Flexbox or Grid for the information cards?"
    A: Either works! Flexbox with flex-wrap is what we've learned. We'll cover Grid later.
  5. Q: "How do I get the image placeholder?"
    A: Use https://via.placeholder.com or find a real image that fits your theme.
  6. Q: "My GitHub Pages isn't showing my CSS. Why?"
    A: Check the file path. Is style.css in the same folder as index.html? Is your link tag correct?
  7. Q: "I'm stuck and don't know where to start."
    A: Start with just the HTML. Don't think about CSS yet. Build the structure first.
  8. Q: "Can I use a CSS framework like Bootstrap?"
    A: No, this project is about learning the fundamentals. Write your own CSS.

QUICK TROUBLESHOOTING GUIDE

If Students Can't Get Started:

  • Sit with them for 2 minutes
  • Have them create index.html and style.css
  • Type the basic boilerplate together
  • Point them to Slide 6 for structure reference

If Someone Finishes Early:

  • Have they deployed to GitHub Pages? If not, do that first
  • Extension: Add hover effects to buttons
  • Extension: Make the nav sticky on scroll
  • Extension: Add subtle animations
  • Extension: Make it responsive (mobile-friendly)

If the Whole Class is Stuck:

  • Stop everyone, pull up the slides
  • Do a 5-minute live demo of the problematic section
  • Answer clarifying questions
  • Resume work time

LANDING PAGE PROJECT GRADING RUBRIC REMINDER

Total: 100 points

  • Structure (25 pts): Proper HTML5 semantic tags, all sections present
  • Styling (40 pts): Closely matches design, appropriate use of Flexbox, colors/fonts match
  • Code Quality (15 pts): Clean, organized code, good naming conventions
  • Deployment (10 pts): Successfully deployed to GitHub Pages, site loads correctly
  • Git Usage (10 pts): Multiple commits with good messages, proper repository setup
Don't expect perfection. Focus on effort and understanding. A "B" project should be recognizable as the landing page design even if it's not pixel-perfect.
