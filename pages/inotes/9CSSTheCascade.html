SLIDES 7-13 8 min

Factor #1: SPECIFICITY

Students fill in the hierarchy (CRITICAL - they must get this order right!):
1. Inline styles - Value: 1000
2. ID selectors - Value: 100
3. Class selectors - Value: 10
4. Type selectors - Value: 1

Important Rule to Emphasize:

1 ID beats 1000 classes
1 class beats 1000 type selectors
Use hand gestures showing hierarchy - hold hand high for inline, lower for each level

Practice Calculations (Slide 12):

What DOESN'T Add (Slide 13):

Universal selector: *
Combinators: >, +, ~, and (space)
SLIDES 9-10 3 min

Specificity Examples Analysis

Example 1 (Two Classes):

Students calculate and determine winner:
Rule 1: .subsection = 10
Rule 2: .main .list = 20
Winner: Rule 2
Why: Two class selectors (20) beats one class selector (10)

Example 2 (ID vs Classes):

Rule 1: #subsection = 100
Rule 2: .main .list = 20
Winner: Rule 1
Why: ID selector (100) beats two class selectors (20)
Show these live in browser - seeing is believing!
SLIDES 14-15 4 min

Factor #2: INHERITANCE

Students list properties that inherit:

Properties That Inherit:

Typography: color, font-family, font-size
Text properties: text-align, text-transform
List properties: list-style

Properties That DON'T Inherit:

Box model: margin, padding, border
Background properties
Position and display
Mnemonic: "Text flows down, boxes don't"

Key Rule:

Direct styling ALWAYS beats inheritance!

Inheritance Example:

Walk through this slowly - it's counterintuitive:
Text color: blue
Why: Direct styling (.child) beats inherited styling even though #parent has higher specificity

Factor #3: RULE ORDER

Rule Order Example:

EMPHASIZE: HTML class order doesn't matter - only CSS file order!
SLIDE 18 2 min

The Complete Cascade Algorithm

Students complete the algorithm steps:
1. Collect all rules targeting the element
2. Check specificity - higher wins!
3. If tied, check direct vs inherited
4. If still tied, last rule wins!
Have class recite this together - repetition helps!
SLIDES 19-21 3 min

Best Practices & DevTools Demo

Best Practices (students fill in):

Do's:
- Keep specificity low
- Use classes for styling (not IDs)
- Organize CSS from general to specific
- Use DevTools to debug cascade issues

Don'ts:
- Avoid inline styles
- Don't use IDs for styling
- Minimize use of !important
- Don't make selectors overly specific

SLIDE 26 2 min

Quick Knowledge Check

Do this as a rapid-fire Q&A with the class
1. Which has higher specificity: .header .nav or #nav?
Answer: #nav (100 > 20)

2. Does margin inherit from parent elements?
Answer: No, margin doesn't inherit

3. If two rules have the same specificity, which wins?
Answer: The last one in the CSS file

4. What's the specificity of div.highlight#main?
Answer: 111 (1 ID + 1 class + 1 type)

Wrap-up & Homework

Key Takeaways (students complete):

1. The cascade resolves conflicts between CSS rules
2. Specificity is calculated by counting IDs, classes, and type selectors
3. Direct styling always beats inheritance
4. When specificity is equal, the last rule wins
5. The best practice is to keep specificity low
