Slides 1-2 Welcome & Agenda 5 min

Students should write their name and date on guided notes header

Talking Points:

  • Welcome students enthusiastically - set positive tone
  • This is a hands-on, project-based course
  • Today's exciting challenge: build a website in 30 minutes!
Build excitement about the ice breaker. Many students are nervous on day 1 - the quick win of deploying a site helps build confidence.

Slides 3-4 Ice Breaker Setup 5 min

Students fill in: "PARTNER'S", "whiteboard", "Professor Reed", "SECRET", "swap"

Setup Instructions:

  • Have whiteboards/markers ready at front of room
  • Students write topics on whiteboard
  • YOU approve each topic (keep them simple!)
  • Students find partners and swap
Show example topics on board: "Coffee Shop", "Space Cats", "Pizza Reviews", "Gym Tips"
Key Phrase to Emphasize: "Perfect is the enemy of DONE!"
Move quickly through pairing. If odd number of students, you can be someone's partner or have a group of 3.

Slides 5-6 Phase 1: Project Setup During 30-min challenge

Students fill in Git commands: "mkdir", "cd", "init"
Git Commands (WRITE ON BOARD):
mkdir partner-topic-site
cd partner-topic-site
git init
Live demo the GitHub repo creation process on projector
VS Code shortcut: "!" for HTML boilerplate
CSS link goes in: "<head>"
JavaScript goes before: "</body>"
Walk around the room during this phase. Many students will need help with GitHub authentication.

Common Issues to Watch For:

  • GitHub authentication problems
  • Forgetting to link CSS/JS files
  • Wrong file names (must be index.html)

Slide 7 Phase 2: Rapid Development During 30-min challenge

Time limit: "30" minutes (but this phase is ~15 minutes of it)

Required Elements Checklist:

  • Meaningful <title> tag
  • Header with site name
  • At least 3 content sections
  • Navigation (even if single page)
  • Footer with your name
Announce time remaining at: 20 min, 10 min, 5 min, 2 min, 30 seconds
Keep reminding: "Ship it! Don't perfect it!"

Slide 8 Phase 3: GitHub Pages Deployment During 30-min challenge

Git commands: "add", "commit", "branch", "push"
Settings location: "Settings"
Deploy from: "branch"
Branch: "main", folder: "/" or "root"
Wait time: "2-3" minutes
Git Push Commands (WRITE ON BOARD):
git add .
git commit -m "Initial site about [topic]"
git branch -M main
git push -u origin main
Show GitHub Pages settings on projector - walk through each step slowly
Have students help each other if they finish early. Builds community!

Slide 9 Phase 4: Custom Domain Demo only - 2 min

DO NOT LET STUDENTS ATTEMPT THIS ALONE - Demo only or do together later
Students should just copy the A record IPs for reference

A Records (Write on board):

185.199.108.153
185.199.109.153
185.199.110.153
185.199.111.153
Explain this is optional and can be done in office hours if interested

Slide 10 Start the Challenge! 30 min total

START THE TIMER NOW - 30:00 countdown

During the Challenge:

  • Walk around constantly
  • Encourage quick decisions
  • Help with technical blocks only
  • Announce time remaining frequently
  • Keep energy high!
Play upbeat background music if appropriate. Creates energy and masks individual conversations.

Slides 11-12 The Odin Project Introduction 3 min

Students fill in: "free", "web", "theodinproject"

Key Points to Emphasize:

  • FREE resource - no textbook needed!
  • Project-based - build real things
  • Industry-relevant - what employers want
  • Active community - Discord for help
Foundations covers:
- HTML: Semantic markup, forms, accessibility
- CSS: Flexbox, Grid, responsive design
- JavaScript: DOM, events, problem solving
- Tools: Git, GitHub, command line
Navigate to theodinproject.com on projector - show the Foundations path

Slides 13-14 Getting Started 2 min

Website: "theodinproject.com"

Learning Path Order:

  1. Introduction - How course works, mindset
  2. Prerequisites - Computer basics, installations
  3. Git Basics - Version control
  4. HTML & CSS - Building pages
  5. JavaScript - Interactivity
Mention that we'll go deeper than Odin in some areas, and Odin is supplementary

Slides 15-16 Thursday's Homework 3 min

STUDENTS MUST COMPLETE ODIN SETUP BEFORE THURSDAY

Homework Components (Have students check boxes):

1. Odin Introduction Sections:

  • How This Course Will Work
  • Introduction to Web Development
  • Motivation and Mindset
  • Asking For Help
  • Join the Odin Community

2. Setup Requirements:

  • Discord setup
  • GitHub setup
  • Tool installations

3. One-Page Website:

  • Finish incomplete tasks
  • Add 3+ sections
  • Style with CSS
  • Add JavaScript
Tell them to post their site URLs in Discord - creates accountability and community

Slides 17-18 Resources & Success Tips 2 min

Success tips blanks: "Day", "Others", "Error", "Documentation", "Projects", "Questions"
Success Tips (EMPHASIZE THESE):
1. Code Every DAY
2. Help OTHERS
3. Read ERROR Messages
4. Read DOCUMENTATION
5. Build PROJECTS
6. Ask QUESTIONS

Resources to Write on Board:

  • theodinproject.com
  • Course website: [your URL]
  • Discord: [invite link]
  • Office hours: [your hours]
  • Email: [your email]

Slides 19-20 Wrap-up & Questions 2 min

Closing Checklist:

  • Congratulate them on deploying their first site!
  • Remind about Thursday homework
  • Answer any questions
  • Share your contact info
  • End with enthusiasm!
Have students fill out reflection questions in guided notes if time permits, or assign as homework

Reflection Questions (for guided notes):

  1. What was most challenging about building the website?
  2. What are you most excited to learn?
  3. What questions do you still have?

After Class Checklist

  • Check that all students joined Discord
  • Review submitted GitHub repos
  • Send welcome email with reminders
  • Post in Discord with encouragement
  • Prepare Thursday's materials

Common First-Day Issues to Address:

  • Students who couldn't deploy - offer office hours help
  • GitHub authentication issues - send setup guide
  • Missing tools/software - provide installation links

Final Teaching Notes

MOST IMPORTANT: Keep energy high and celebrate small wins!

Remember:

  • This sets the tone for the entire semester
  • Focus on building confidence, not perfection
  • Every student should leave with a deployed site
  • Create a supportive, collaborative environment
  • Your enthusiasm is contagious!
Take a photo of the class with their sites displayed - great for social media and building class community!
