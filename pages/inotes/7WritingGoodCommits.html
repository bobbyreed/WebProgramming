PART 1: WRITING GOOD COMMITS (Slides 1-10)

Slides 1-2: Introduction & Agenda (2 min)
Quick Start:
  • Welcome students, ensure they have guided notes
  • Emphasize Recipe Project due tomorrow
  • Today is hands-on - they'll leave with progress on project
Slide 3: Why Commit Messages Matter (3 min)
GUIDED NOTES ANSWERS - Students fill in 3 reasons:
  1. Job Applications - Employers look at GitHub
  2. Team Collaboration/Bug Tracking
  3. Future reference (6 months later)
Key Points to Emphasize:
  • Tell story about reviewing candidate's GitHub with commits like "asdfasdf"
  • This is the EASIEST way to stand out as a junior developer
  • STRESS: "commit history" for notes blank
Slide 4: Bad Commit Examples (2 min)
Read each bad example out loud with dramatic effect!
  • Ask: "What's wrong with 'fix bug'?" - Wait for responses
  • Point out: No context, no why, unprofessional
  • Mention: These are REAL examples from student projects
Slide 5: Anatomy of Good Commit (3 min)
GUIDED NOTES ANSWERS:
  • Subject Line: WHAT you did
  • Character limit: 50-72 characters
  • Body: WHY you did it and HOW it solves
  • Leave a blank line between subject and body
Walk through the example slowly:
  • Subject: Clear, specific action
  • Blank line (emphasize this!)
  • Body: Context and reasoning
Slide 6: Bad vs Good Comparison (2 min)
GUIDED NOTES - Students analyze:

Bad problems: No context, vague, unclear what was fixed

Good improvements: Specific, explains issue, provides context

Slide 7: When to Commit (2 min)
Video game save point analogy works great!
  • Have students check ALL boxes in guided notes
  • Emphasize: "Commit early and often"
  • Warning about broken code - use branches or WIP
Slide 8: The 7 Rules (3 min)
GUIDED NOTES ANSWERS - Fill in blanks:
  1. Separate with blank line
  2. Limit to 50-72 characters
  3. Capitalize the subject line
  4. Don't end with period
  5. Use imperative mood
  6. Wrap at 72 characters
  7. Explain what and why
Test tip: "If applied, this commit will..." - Write on board!
Slide 9: Multi-line Commits (2 min)
GUIDED NOTES ANSWERS:
  • Command: git commit (no -m flag)
  • Opens: VS Code
QUICK DEMO: Show this live if possible - open VS Code with git commit
Slide 10: Practice Exercise (2 min)
PRACTICE ANSWERS - Example improvements:
  • "css changes" ‚Üí "Update CSS grid layout for recipe cards"
  • "oops" ‚Üí "Revert accidental deletion of index.html"
  • "FINALLY!!!!" ‚Üí "Fix image path preventing photos from loading"
  • "stuff" ‚Üí "Add navigation links between recipe pages"
Have 2-3 students share their improvements

PART 2: RECIPE PROJECT (Slides 11-24)

Slides 11-12: Project Requirements (3 min)
GUIDED NOTES ANSWERS:
  • Main index page
  • At least 3 recipe pages
  • Folders for recipes and images
  • Links between all pages
  • Title using h1 tag
  • Image with alt attribute
  • Description in p tag
  • Ingredients: ul (unordered)
  • Steps: ol (ordered)
  • Link back to main/index page

TROUBLESHOOTING GUIDE

Common Problem Quick Solution Teaching Moment
Git not initialized git init in project folder Always start with git init
Images broken Check: ../ paths, file extensions, case Relative vs absolute paths
Links not working Add .html, check folder structure File paths in HTML
Can't push to GitHub Check remote: git remote -v Local vs remote repositories
VS Code won't open for commit Use git commit -m as backup Git editor configuration
