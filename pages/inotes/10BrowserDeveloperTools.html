SLIDE 2: Why Browser DevTools? 2 min

Instructor Notes:

  • STORY: Share a personal debugging story where DevTools saved hours
  • Emphasize this is used DAILY in professional development
  • "You'll spend more time in DevTools than in your code editor some days!"

GUIDED NOTES EMPHASIS

Students should list the 6 main uses:

  1. Debug HTML/CSS
  2. Experiment without saving
  3. Inspect other sites
  4. Measure spacing/colors
  5. Test responsive designs
  6. Check performance
Key Quote to Emphasize:
"DevTools turns you from a guesser into a detective. No more 'why isn't this working?' - you'll know EXACTLY why."

SLIDE 4: Three Ways to Open DevTools 2 min

Instructor Notes:

  • DEMONSTRATE ALL THREE METHODS LIVE
  • Have students follow along with each method
  • Make them close and reopen DevTools 3 times

GUIDED NOTES EMPHASIS

Priority order to write down:

  1. Right-click ‚Üí Inspect (most common, targets element)
  2. F12 (fastest when you know it)
  3. Ctrl/Cmd+Shift+I (universal shortcut)
Common Issues:
  • Mac users: Remind about Cmd instead of Ctrl
  • F12 might be disabled on some laptops (use Fn+F12)

SLIDE 6: Demo 1 - Basic Element Inspection 5 min

LIVE DEMO INSTRUCTIONS

  1. First: Right-click the purple demo box
  2. Show: How DevTools jumps to that element
  3. Highlight: The correlation between page and HTML
  4. Demonstrate: Hovering shows dimensions
  5. Let students: Try it themselves for 2 minutes

GUIDED NOTES EMPHASIS

Students should write the inspection workflow:

  1. Right-click element
  2. Select "Inspect"

Talking Points:

  • "The HTML might look different than what you wrote - browser adds things"

SLIDE 7: The Element Selector Tool 3 min

LIVE DEMO INSTRUCTIONS

  1. Click the selector icon (top-left of DevTools)
  2. Hover over various elements slowly
  3. Show the dimension overlay
  4. Click to select
  5. Press Escape to exit selector mode

GUIDED NOTES EMPHASIS

Shortcut to memorize: Ctrl+Shift+C (Cmd+Shift+C on Mac)

Pro Tip to Share: "This is how you inspect things you can't right-click, like dropdown menus!"

SLIDE 8: Understanding the DOM Tree 3 min

Instructor Notes:

  • Explain DOM = Document Object Model = "live HTML structure"
  • Show expanding/collapsing nodes
  • Demonstrate that changes here affect the page immediately

QUICK DEMO

  1. Double-click text to edit it
  2. Drag an element to reorder
  3. Right-click ‚Üí Delete to remove element
  4. Refresh to show changes are temporary

GUIDED NOTES EMPHASIS

Key concept: "DOM changes are TEMPORARY - refresh = gone"

SLIDE 9: The Styles Panel 4 min

THIS IS CRITICAL - SLOW DOWN HERE

GUIDED NOTES EMPHASIS

Students MUST understand this hierarchy:

  1. element.style {} = Inline styles (highest priority)
  2. .class-name {} = CSS rules that apply
  3. Strikethrough = Overridden styles (still there, just losing)
  4. inherited = From parent elements

LIVE DEMO INSTRUCTIONS

  1. Select a heading element
  2. Show inherited font from body
  3. Show class styles
  4. Show overridden styles with strikethrough
  5. Explain the "cascade" briefly
Key Analogy: "Strikethrough is like a losing argument - the rule tried to apply but got overruled by something stronger"

SLIDE 10: Demo 2 - Live Style Editing 5 min

EXTENDED DEMO TIME - MOST IMPORTANT DEMO

Step-by-Step Demo

  1. Inspect the blue demo box

GUIDED NOTES EMPHASIS

Instructor Notes:

SLIDE 13: Demo 3 - Real-World Debugging 5 min

DEBUGGING CHALLENGE DEMO

The Setup: "Here's a broken layout - let's debug it together!"

Walk Through:
  1. Problem 1: Heading is red but CSS says #333
    • Find the !important in inline styles
    • Explain why !important wins
  2. Problem 2: Text too small
    • Show inline style overriding stylesheet
  3. Problem 3: Button not purple
    • Show inline gray overriding class

GUIDED NOTES EMPHASIS

Debugging process:

  1. Inspect element
  2. Look for strikethrough
  3. Find winning rule
  4. Check for !important
  5. Check specificity

SLIDE 15: Computed Styles Tab 2 min

Instructor Notes:

SLIDE 16: Console & document.querySelector 3 min

CONSOLE DEMO

  1. Open Console tab
  2. Type: document.querySelector('h1')
  3. Show the element reference
  4. Type: $0.style.color = 'red'
  5. Show $0 is selected element
  6. Show $ and $$ shortcuts

GUIDED NOTES EMPHASIS

Console shortcuts:

  • $0 = Currently selected element
  • $('selector') = querySelector
  • $$('selector') = querySelectorAll

SLIDE 24: Q&A 3-5 min

Common Questions & Answers:

Q: "How do I save my changes?"
A: Copy the CSS from DevTools and paste into your CSS file

Q: "Why can't I inspect some elements?"
A: Some sites block right-click; use Ctrl+Shift+C instead

Q: "Can I break the website?"
A: Only for you, temporarily - refresh fixes everything

Common Problems & Solutions

Problem: "DevTools won't open"

  • Check if F-keys are locked (laptop)
  • Try right-click method
  • Check if browser extensions blocking

Problem: "Changes aren't saving"

  • Remind: DevTools changes are temporary
  • Show how to copy CSS

Problem: "Can't find Styles panel"

  • Might be collapsed - show resizing
  • Might be in wrong tab - show Elements tab
