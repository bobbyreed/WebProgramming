SLIDES 1-2: Introduction & Learning Objectives

Opening Statement:

"Today we're going behind the scenes of something you use every day - the web. By the end of this class, you'll understand what actually happens when you type google.com and hit enter."

Start with energy! This sets the tone for the entire lecture. Consider asking: "Who can guess how many steps happen between typing google.com and seeing the page?"

Key Points to Emphasize:

  • This is foundational knowledge for everything we'll build
  • We're going BROAD today, not deep
  • Have guided notes ready - we'll be filling in blanks together

SLIDES 3-5: Networks and Internet Basics

Slide 3: What's a Network?

Students fill in: A network is TWO or more computers connected together to share RESOURCES

Use analogy: "Your home WiFi is like a small neighborhood, the campus network is like a city, and they all connect to form..."

Slide 4: The Internet

Students fill in: The Internet is a GLOBAL SYSTEM of NETWORKS. It's the INFRASTRUCTURE that carries data.
EMPHASIZE: Internet ‚â† World Wide Web. Use highway (Internet) vs cars (services like Web) analogy.

Slide 5: Key Players

Students complete table:
  • ISP: Connects you to the internet / Provides internet access
  • Router: Directs traffic between networks
Students often confuse routers and modems. Clarify: Modem = translator (coax/fiber to ethernet), Router = traffic director
3 min

SLIDE 6: Packets

Essential Concept:

Book shipping analogy: "Imagine shipping a encyclopedia - would you send the whole thing or individual pages?"

Students fill in packet contents:
  1. SOURCE address
  2. DESTINATION address
  3. Piece of the DATA
  4. SEQUENCE number
Practice question answer: Efficiency (multiple routes), Reliability (resend only lost packets), Speed (parallel paths)
Draw on board: Show packets taking different routes to same destination. This visual really helps!
4 min

SLIDES 7-8: IP Addresses and DNS

Slide 7: IP Addresses

An IP address is like a PHONE number for computers
  • IPv4: 4 numbers (0-255)
  • IPv6: Much longer, HEXADECIMAL format

Transition question: "But who memorizes IP addresses? Nobody! That's why we need..."

Slide 8: DNS

DNS is the internet's PHONE book
It translates DOMAIN names into IP addresses
DNS Process blanks:
  1. You type: www.okcu.edu (or any domain)
  2. DNS Server looks up the IP ADDRESS
  3. Returns: 104.16.20.146 (example)
  4. Your browser connects to that IP/ADDRESS
Use real example: "Let's look up okcu.edu's IP right now!" (Use nslookup or online tool)
2 min

SLIDE 9: Client-Server Model

Restaurant Analogy:

"Client is the customer, Server is the waiter/kitchen. Client orders (requests), Server delivers (responds)."

Table completion:
  • Client: REQUESTS information, Your web BROWSER
  • Server: PROVIDES information, Hosts WEBSITES
Example interaction:
Client says: "Give me the homepage" or "Send me index.html"
Server responds: "Here's the HTML, CSS, and JavaScript"
2 min

SLIDE 10: Web Components - Critical Distinctions!

Students ALWAYS confuse these. Spend time here!
Definitions to complete:
  1. Web Browser: Software to access/view websites
    Examples: Chrome, Firefox, Safari
  2. Search Engine: Website that finds other websites
    Examples: Google, Bing, DuckDuckGo
  3. Web Page: A single document/file on the web
  4. Web Server: Computer that hosts and serves web pages
Quick check: "Is Google Chrome a search engine?" (NO! It's a browser) "Is Google.com a browser?" (NO! It's a search engine website)
5 min

SLIDES 11-14: The Google Search Journey ‚≠ê

This is the MOST IMPORTANT section - it ties everything together!

Setup (Slide 11):

"Now let's use everything we've learned to trace what happens when you search Google..."

The Journey (Slides 12-14):

Students fill in steps:
  1. Browser checks its CACHE
  2. If not found, browser asks DNS for IP address
  3. DNS returns: 142.250.80.46 (example)
  4. Browser establishes CONNECTION with Google's server
  5. Google sends back HTML, CSS, and JavaScript
  6. Query is ENCRYPTED (HTTPS) and sent
  7. Google searches its BILLIONS OF INDEXED pages
  8. Results returned in about 0.5 seconds!
Draw this on the board as you go! Visual learners need to SEE the path of data.
Pause after each step: "So far we have DNS lookup... then what? Who remembers about packets?"
1 min

SLIDE 15: HTTPS Security

Quick Security Lesson:

HTTP vs HTTPS comparison:
  • HTTP: UNENCRYPTED, Like POSTCARDS
  • HTTPS: ENCRYPTED, Like SEALED ENVELOPES
  • HTTP: Anyone can READ
  • HTTPS: Only recipient can READ/DECRYPT
When to look for HTTPS: When entering passwords, payment info, or any sensitive data
Quick check: "Everyone look at your browser right now - do you see the padlock?"
2 min

SLIDE 16: Browser Check Activity

Have students visit whatismybrowser.com NOW
Students record: Browser name and version

Why This Matters:

  • Different browsers = different rendering engines
  • You'll need to test in multiple browsers
  • Chrome DevTools vs Firefox Developer Edition
"Surprised by your version number? Browsers update automatically now - that's why version numbers are so high!"
1 min

SLIDE 17: Internet vs. World Wide Web

Students complete comparison:
  • Internet: The INFRASTRUCTURE, Physical NETWORKS, Since 1960s
  • WWW: One SERVICE on internet, Web pages with LINKS, Created 1989, Uses HTTP/HTTPS protocol
Analogy completion: The Web runs ON the Internet, like cars on a highway / apps on a phone / trains on tracks
"Email existed before the Web! So did file transfer (FTP). The Web is just one way to use the Internet."

‚ö†Ô∏è Common Student Confusions & Clarifications

Common Confusion Clarification to Provide
Browser = Search Engine Browser is the APP (Chrome), Search Engine is a WEBSITE (google.com)
Internet = WiFi WiFi is just how you connect wirelessly to your LOCAL network
One packet = one file Files are broken into MANY packets, each takes its own route
DNS = Google DNS translates names to IPs. Google is a search engine that finds websites
Server = Big Computer Server is a ROLE, not a size. Your laptop can be a server!

üéØ Key Takeaways Students MUST Understand

  1. The Internet is infrastructure - physical networks connected globally
  2. Data travels in packets - small chunks that can take different routes
  3. DNS translates names to numbers - like a phone book for the internet
  4. Client requests, Server responds - the fundamental web interaction
  5. Browser ‚â† Search Engine - one is software, one is a website
  6. HTTPS = Secure - always look for the padlock
  7. Internet ‚â† World Wide Web - the web is just one service on the internet
If students leave understanding these 7 points, the lecture was successful!
