OPENING (5 minutes)

SLIDES 1-3 Introduction & Learning Objectives

Start with a visual: Show a poorly laid out webpage vs a well-structured one. Ask: "What makes the difference?"

Key Points to Cover:

  • Review box model from last class (every element is a box)
  • Today: How boxes interact with each other
  • This is fundamental - everything in CSS layout builds on this
Students should fill in their name and date on guided notes

NORMAL FLOW (5 minutes)

SLIDES 4-5 Understanding Normal Flow

Students fill in: "the default way browsers lay out HTML elements without any CSS positioning"
Key Concepts Answers:
1. Elements render in HTML order
2. Block elements stack vertically
3. Inline elements flow horizontally
4. No overlapping unless positioned
Show a simple HTML file with NO CSS - demonstrate that elements still have default display behaviors

DISPLAY PROPERTY (3 minutes)

SLIDE 6 The Display Property

Students fill in display values:
- display: block; /* Elements stack vertically */
- display: inline; /* Elements flow horizontally */
- display: inline-block; /* Best of both worlds */
- display: none; /* Hides the element completely */
Emphasize that we can CHANGE any element's default display - this is powerful!

BLOCK ELEMENTS (8 minutes)

SLIDES 7-9 Block Elements Deep Dive

Students check these characteristics for block elements: ‚úì Start on a new line ‚úì Take up full width available ‚úì Stack vertically ‚úì Can have width and height set ‚úì All margin/padding sides work
Common Block Elements:
Structural: div, section, article, header, footer, main, nav
Headings: h1, h2, h3, h4, h5, h6
Content: p, blockquote, pre
Lists: ul, ol, li
Live code three divs with different background colors. Show how they stack and take full width even with small content.
Block elements are like paragraphs in a document - each starts on its own line!

INLINE ELEMENTS (8 minutes)

SLIDES 10-12 Inline Elements Deep Dive

Students check these characteristics for inline elements: ‚úì Flow with text content ‚úì Only take necessary space (Leave unchecked: start on new line, take full width, can set width/height, etc.)
Common Inline Elements (any 6):
span, a, strong, em, img, button, input, label, code, small, br, cite
Create a paragraph with multiple inline elements (span, strong, em, a). Show how they flow within the text.

SLIDES 13-14 ‚ö†Ô∏è CRITICAL CONCEPT: Inline Padding/Margin

SPEND TIME HERE! This is a common source of confusion and bugs.
Students fill in: "don't push other elements away / can cause overlapping"
MUST DEMONSTRATE: Add large vertical padding to a span. Show how it overlaps lines above and below!
Tell students: "If you remember ONE thing about inline elements, remember that vertical spacing doesn't work normally!"

COMPARISON (5 minutes)

SLIDE 13 Block vs Inline Comparison Table

Students complete comparison table:
Table Answers:
Line breaks? Block: Yes (before & after) | Inline: No
Default width: Block: 100% | Inline: Content-based
Can set width/height? Block: Yes | Inline: No
Vertical margin/padding: Block: Works normally | Inline: Problematic

INLINE-BLOCK (5 minutes)

SLIDES 15-17 The Middle Ground

Students fill in:
From inline: "doesn't break lines / flows horizontally"
From block: "respects all box model properties / can set dimensions"
Create navigation with links. First show as inline (spacing problems), then as inline-block (perfect!).
Mention that while inline-block is useful, flexbox (next lesson) is often better for modern layouts.

DIVS AND SPANS (5 minutes)

SLIDES 18-20 Generic Containers

Students complete div/span table:
Default display: div = block | span = inline
Semantic meaning: Both = none
Common use: div = layout/grouping | span = text styling
Always use semantic HTML first! Only use div/span when no semantic element fits.
Best Practice: "no other semantic element applies"

PRACTICAL EXAMPLES (5 minutes)

SLIDES 21-23 Real-World Applications

Live code a navigation bar and a card component. Show display property changes in real-time.
Reference their Recipe Projects - they'll be adding these concepts to style them!

COMMON GOTCHAS & QUIZ (5 minutes)

SLIDES 24-25 Troubleshooting & Quick Check

Quiz Answers for Practice Problems:
Display Type Answers:
1. Navigation links: inline-block
2. Main container: block
3. Bold text in paragraph: inline
4. Cards in layout: inline-block (or flex)
5. Page header: block

Fix the CSS: display: inline-block;

Output Lines: 3 lines
Explanation: First div (line 1), both spans on same line (line 2), last div (line 3)
Have students predict the answer BEFORE revealing. Use hand voting or chat responses.

COMMON STUDENT QUESTIONS

  1. "Why doesn't my padding work on spans?"
    ‚Üí Inline elements don't respect vertical spacing normally. Use inline-block.
  2. "Can I make any element block or inline?"
    ‚Üí Yes! But respect semantic meaning - a paragraph is still a paragraph even if displayed inline.
  3. "When should I use inline-block vs flexbox?"
    ‚Üí Inline-block for simple horizontal layouts, flexbox for complex layouts (next class!).
  4. "Why are there gaps between my inline-block elements?"
    ‚Üí HTML whitespace creates gaps. Solutions: no whitespace, font-size: 0 on parent, or use flexbox.
