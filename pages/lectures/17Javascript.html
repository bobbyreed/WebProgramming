Historical Story #1: JavaScript Created in Just 10 Days

  • May 1995: 34-year-old Brendan Eich created the first version of JavaScript in just 10 days
  • Netscape management gave Eich an almost impossible deadline to create a new programming language for beginning programmers
  • Originally code-named "Mocha", then renamed "LiveScript" in September 1995
  • Finally renamed "JavaScript" in December 1995 in a joint announcement with Sun Microsystems
üìö Source: The New Stack - "Brendan Eich on Creating JavaScript in 10 Days"

Why This Matters

üîó Connection to Your Learning:

This explains why JavaScript has some quirks but also incredible flexibility. The 10-day timeline meant some design decisions were made quickly, but it also gave JavaScript the adaptability that helped it evolve into the powerful language it is today!

Historical Story #2: JavaScript's Name Was Just Marketing

  • The name "JavaScript" was purely a marketing decision
  • Technically, JavaScript had little in common with Java
  • At the time, Java was the hot language, and Netscape wanted to capitalize on that popularity
  • Many people still confuse JavaScript with Java, even though they're completely different languages with different purposes and designs
üìö Source: The History of JavaScript

Why This Matters

üîó Connection to Your Learning:

Understanding JavaScript as its own unique language, not a subset of Java, is crucial. Don't let the name fool you! JavaScript has its own syntax, design philosophy, and ecosystem. Learning JavaScript gives you skills distinct from Java programming.

Working with Numbers üî¢

Basic Math Operations

let x = 10;
let y = 5;

console.log(x + y);  // 15 (Addition)
console.log(x - y);  // 5  (Subtraction)
console.log(x * y);  // 50 (Multiplication)
console.log(x / y);  // 2  (Division)
console.log(x % y);  // 0  (Modulus - remainder)
console.log(x ** y); // 100000 (Exponentiation)
Remember PEMDAS! JavaScript follows standard order of operations.

Working with Strings üìù

Three Ways to Write Strings

String Concatenation

Useful String Methods üõ†Ô∏è

let text = "Hello, JavaScript!";

console.log(text.length);           // 18
console.log(text.toLowerCase());    // "hello, javascript!"
console.log(text.toUpperCase());    // "HELLO, JAVASCRIPT!"
console.log(text.includes("Java")); // true
console.log(text.slice(0, 5));      // "Hello"
console.log(text.replace("Hello", "Hi")); // "Hi, JavaScript!"
Pro Tip: Strings are immutable! Methods return new strings without changing the original.

JavaScript Operators ‚öôÔ∏è

Assignment Operators

let x = 5;
x += 3;  // x = x + 3 ‚Üí 8
x -= 2;  // x = x - 2 ‚Üí 6
x *= 2;  // x = x * 2 ‚Üí 12
x /= 4;  // x = x / 4 ‚Üí 3

Comparison Operators

5 == "5"   // true (loose)
5 === "5"  // false (strict)
5 != "5"   // false
5 !== "5"  // true
5 > 3      // true
5 <= 5     // true
Important: Always use === and !== instead of == and !=

Increment & Decrement ‚ûï‚ûñ

Shorthand for Adding/Subtracting 1

let score = 10;

score++;  // score = score + 1 ‚Üí 11
score--;  // score = score - 1 ‚Üí 10

// They're the same as:
score = score + 1;
score = score - 1;

Common Use Cases

  • Counting items
  • Loop counters
  • Score tracking
  • Page navigation

When to Use const vs let ü§î

Use const for:

  • Values that won't change
  • Constants (PI, MAX_SIZE)
  • Function declarations
  • Objects/Arrays (we'll learn soon)
const PI = 3.14159;
const MAX_USERS = 100;
const API_KEY = "abc123";

Use let for:

  • Values that will change
  • Counter variables
  • Loop variables
  • Reassignments
let score = 0;
let userName = "Guest";
let isLoggedIn = false;
Rule of Thumb: Start with const. Only use let when you know you'll need to reassign!

Type Coercion üîÑ

JavaScript will automatically convert types. This can be helpful... or confusing! 
console.log("5" + 5);    // "55" (string)
console.log("5" - 5);    // 0 (number)
console.log("5" * "2");  // 10 (number)
console.log(true + 1);   // 2 (boolean ‚Üí number)

// Be careful!
console.log(5 == "5");   // true (coerces types)
console.log(5 === "5");  // false (strict check)
This is why we use ===! It checks both value AND type.
