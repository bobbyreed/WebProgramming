What Are Functions? üéÅ

Reusable Code Packages

Without Functions

  • Write same code multiple times
  • Hard to maintain
  • Prone to errors
  • Messy and repetitive

With Functions

  • Write once, use many times
  • Easy to update
  • Less bugs
  • Clean and organized

Your First Function üëã

Let's Say Hello!

// Defining a function
function sayHello() {
  console.log("Hello, World!");
}

// Calling (using) the function
sayHello(); // Output: Hello, World!
sayHello(); // Output: Hello, World!
sayHello(); // Output: Hello, World!
Notice: We write the function once but can call it many times!

Anatomy of a Function üî¨

function functionName(parameters) {
  // Function body
  // Code to execute
  return result;
}

Parts Explained:

  • function - Keyword to declare a function
  • functionName - What you call your function
  • parameters - Input values (optional)
  • {...} - Function body (the code)
  • return - Send a value back (optional)

Parameters vs Arguments üì•

// PARAMETER: placeholder in function definition
function favoriteAnimal(animal) {
  return animal + " is my favorite animal!";
}

// ARGUMENT: actual value passed when calling
console.log(favoriteAnimal('Goat'));
// Output: Goat is my favorite animal!

Remember:

üìå Parameter = Placeholder (in definition)

üìå Argument = Actual value (when calling)

Multiple Parameters ‚ûï

Functions Can Take Many Inputs!

function greetPerson(firstName, lastName, age) {
  return `Hello, ${firstName} ${lastName}! You are ${age} years old.`;
}

console.log(greetPerson('John', 'Doe', 25));
// Output: Hello, John Doe! You are 25 years old.
Order Matters! Arguments must match parameter order

Return Values üì§

Getting Data Back from Functions

Without Return

function add(a, b) { console.log(a + b); } let result = add(5, 3); console.log(result); // undefined

With Return

function add(a, b) { return a + b; } let result = add(5, 3); console.log(result); // 8

Return Stops Execution üõë

function checkAge(age) {
  if (age < 18) {
    return "Too young!";
  }
  return "Welcome!";
  console.log("This never runs!"); // Unreachable code
}

console.log(checkAge(16)); // "Too young!"
console.log(checkAge(21)); // "Welcome!"
Key Point: Nothing after a return statement executes!

Function Scope üîí

Variables Have Boundaries!

let globalVar = "I'm global!";

function myFunction() {
  let localVar = "I'm local!";
  console.log(globalVar); // ‚úÖ Works!
  console.log(localVar); // ‚úÖ Works!
}

myFunction();
console.log(globalVar); // ‚úÖ Works!
console.log(localVar); // ‚ùå Error! localVar is not defined

Rule: Variables inside functions stay inside functions!

Arrow Functions ‚û°Ô∏è

Modern Function Syntax

// Traditional function
function add(a, b) {
  return a + b;
}

// Arrow function (same thing!)
const add = (a, b) => {
  return a + b;
};

// Even shorter (implicit return)
const add = (a, b) => a + b;
Note: Arrow functions are optional for now - focus on regular functions first!

Default Parameters üéØ

Providing Fallback Values

// Without default parameters
function greet(name) {
  if (!name) {
    name = "friend";
  }
  return `Hello, ${name}!`;
}

// With default parameters (easier!)
function greet(name = "friend") {
  return `Hello, ${name}!`;
}

console.log(greet()); // "Hello, friend!"
console.log(greet("Alice")); // "Hello, Alice!"

The Call Stack üìö

How Functions Execute

function first() {
  console.log("First start");
  second();
  console.log("First end");
}

function second() {
  console.log("Second");
}

first();

Output Order:

  1. "First start"
  2. "Second"
  3. "First end"

Why? Functions pause to complete inner function calls!

Exercise 1: add7

Write a function that takes one number and returns that number + 7

function add7(num) { // Your code here! } console.log(add7(5)); // Should output: 12

function add7(num) {

}

  • Returns the result (important!)
  • Works with any number

Real World Functions üåç

Functions You'll Write Often

// Form validation
function isValidEmail(email) {
  return email.includes('@') && email.includes('.');
}

// Format currency
function formatPrice(price) {
  return '$' + price.toFixed(2);
}

// Calculate age
function calculateAge(birthYear) {
  const currentYear = new Date().getFullYear();
  return currentYear - birthYear;
}

Function Best Practices ‚≠ê

‚úÖ DO:

  • Use descriptive names (calculateTotal not calc)
  • Keep functions small and focused (one task)
  • Return values instead of just console.log
  • Use parameters instead of global variables

‚ùå DON'T:

  • Make functions too complex (if > 20 lines, split it)
  • Use vague names (doStuff, thing)
  • Forget to return values when needed
  • Modify global variables inside functions
