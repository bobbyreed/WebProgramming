What Are Objects? ü§î

Objects are collections of related data and functionality

// A simple object representing a person const person = { name: "Alice", age: 25, isStudent: true, greet: function() { return "Hello, I'm " + this.name; } };

Arrays

Ordered lists of values

Key-value pairs

Creating Objects üèóÔ∏è

Object Literal Syntax

// Empty object const emptyObj = {}; // Object with properties const car = { make: "Toyota", model: "Camry", year: 2024, isElectric: false, features: ["GPS", "Bluetooth", "Backup Camera"] }; // Properties can be any data type!
Pro Tip: Use meaningful property names that describe the data they hold. Objects can contain primitives, arrays, and even other objects!

Accessing Object Properties üîë

Two Ways to Access

Dot Notation

const user = { name: "Bob", email: "bob@example.com" }; console.log(user.name); // "Bob" user.age = 30; // Add property

Bracket Notation

const user = { name: "Bob", email: "bob@example.com" }; console.log(user["name"]); // "Bob" const prop = "email"; console.log(user[prop]); // Dynamic!
When to use which?
  • Dot notation: When you know the property name
  • Bracket notation: For dynamic properties or special characters

Object Methods ‚ö°

Functions stored as object properties are called methods

const calculator = { value: 0, add: function(num) { this.value += num; return this.value; }, reset: function() { this.value = 0; }, getValue: function() { return this.value; } }; calculator.add(5); // 5 calculator.add(3); // 8 calculator.reset(); // value is now 0
Important: The keyword this refers to the object that owns the method!

Primitives vs Objects: The Fundamental Difference

Primitives (Copy by Value)

  • Numbers
  • Strings
  • Booleans
  • null, undefined

Store actual values

Objects (Copy by Reference)

  • Objects {}
  • Arrays []
  • Functions
  • Dates

Store references to values

This is THE most important concept today! Understanding this will save you from countless bugs.

Value vs Reference Visualized üëÅÔ∏è

Primitives (Copies)

let x = 10; let y = x; // y gets a COPY of 10 y = 20; console.log(x); // 10 (unchanged!) console.log(y); // 20

let obj1 = {value: 10}; let obj2 = obj1; // obj2 points to SAME object obj2.value = 20; console.log(obj1.value); // 20 (changed!) console.log(obj2.value); // 20

Reference Behavior in Action üîÑ

// This will surprise you if you don't understand references! const original = { name: "Original", items: ["apple", "banana"] }; const copy = original; // NOT a copy - a reference! copy.name = "Modified"; // Changes original! copy.items.push("orange"); // Changes original's array! console.log(original.name); // "Modified" üò± console.log(original.items); // ["apple", "banana", "orange"]
Real-world analogy: Think of it like sharing a Google Doc link vs. downloading and emailing a copy. Changes to the shared doc affect everyone!

Passing Objects to Functions üì§

function updateUser(user) { user.logged_in = true; // Modifies the original! user.last_active = "now"; } function incrementCounter(count) { count = count + 1; // Doesn't affect original return count; } const myUser = { name: "Alice", logged_in: false }; let myCount = 0; updateUser(myUser); // myUser is modified incrementCounter(myCount); // myCount stays 0 console.log(myUser.logged_in); // true ‚úì console.log(myCount); // 0 (unchanged)
Remember: Objects passed to functions can be modified inside the function! Primitives passed to functions are copied, so changes don't affect the original.

Reassigning vs Mutating üîÄ

A Crucial Distinction

Mutating (Changes Object)

let animal = { species: "dog" }; let pet = animal; animal.species = "cat"; // MUTATE console.log(pet.species); // "cat" // Both variables see the change!

Reassigning (New Object)

let animal = { species: "dog" }; let pet = animal; animal = { species: "cat" }; // REASSIGN console.log(pet.species); // "dog" // pet still points to original!
Key Insight: Reassigning breaks the reference connection. Mutating affects all references to that object.

Common Mistakes to Avoid ‚ö†Ô∏è

Gotcha #1: Thinking you're copying when you're not

const settings = { theme: "dark" }; const backup = settings; // NOT a copy! backup.theme = "light"; // Oops, changed original!

Gotcha #2: const doesn't mean immutable

const user = { name: "Bob" }; user.name = "Alice"; // This works! Object is mutable user = {}; // This fails! Can't reassign const

Gotcha #3: Checking object equality

const obj1 = { x: 1 }; const obj2 = { x: 1 }; console.log(obj1 === obj2); // false! Different objects
