SyntaxError ‚úçÔ∏è

When your code doesn't follow JavaScript grammar

// Missing parentheses
function helloWorld() {
  console.log "Hello World!" // SyntaxError!
}
Uncaught SyntaxError: Unexpected string
Common Syntax Errors:
  • Missing brackets: ( ) { } [ ]
  • Missing quotes: " " or ' '
  • Missing semicolons (sometimes)
  • Extra or missing commas

‚ùå Wrong

‚úÖ Correct

  name: "Alice",
  age: 25

Missing comma between object properties!

TypeError üîß

When you use the wrong type or method

const str1 = "Hello";
const str2 = "World!";
const message = str1.push(str2); // TypeError!
Uncaught TypeError: str1.push is not a function

Why?

.push() is an Array method, not a String method!

Strings use .concat() instead

Common TypeError Scenarios

1. Calling non-functions

const num = 42;
num(); // TypeError: num is not a function

2. Properties on null/undefined

let user;
console.log(user.name); // TypeError!

3. Wrong method for type

const num = 123;
num.toUpperCase(); // TypeError!

Common Beginner Mistakes üéØ

Top 5 Error-Causing Mistakes:

  1. Case sensitivity: myVariable vs myvariable
  2. Missing quotes: class=container vs class="container"
  3. = vs ==: Assignment vs comparison
  4. Scope issues: Variables inside blocks
  5. Async timing: Using data before it loads
Remember: Everyone makes these mistakes - even senior developers!

Handling Errors Gracefully üõ°Ô∏è

Introduction to try...catch

try {
  // Code that might throw an error
  const data = JSON.parse(userInput);
  console.log("Success!", data);
} catch (error) {
  // Handle the error gracefully
  console.error("Invalid JSON:", error.message);
  // Show user-friendly message
  alert("Please enter valid data");
}
When to use: When dealing with user input, API calls, or any operation that might fail
