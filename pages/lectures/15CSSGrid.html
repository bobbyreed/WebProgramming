What is CSS Grid? üìê

CSS Grid is a powerful layout system for creating two-dimensional layouts with rows AND columns!

Fun Fact:

  • üìÖ Started development in 1996 by CSS co-creator Dr. Bert Bos
  • üì∞ Inspired by grid layouts in newspapers and magazines
  • üöÄ Finally introduced to all major browsers in 2017
  • ‚ú® The gap property used to be Grid-only, now works with Flex too!

Grid vs Flexbox ‚öîÔ∏è

Use Grid When:

  • ‚úÖ You need 2D layout (rows AND columns)
  • ‚úÖ Layout comes first (layout-first design)
  • ‚úÖ You want precise control over positioning
  • ‚úÖ Building page-level layouts
  • ‚úÖ Creating complex, overlapping designs

Use Flexbox When:

  • ‚úÖ You need 1D layout (row OR column)
  • ‚úÖ Content comes first (content-first design)
  • ‚úÖ Items should grow/shrink dynamically
  • ‚úÖ Building component-level layouts
  • ‚úÖ Simple alignment tasks
Pro Tip: Grid and Flexbox work great TOGETHER! Use Grid for the overall layout and Flexbox for components.

Setting Up a Grid üèóÔ∏è

Step 1: Create a Grid Container

.container {
  display: grid;
}

That's it! All direct children automatically become grid items.

Remember: Only DIRECT children become grid items, not nested elements.

Defining Columns and Rows

.container {
  display: grid;
  grid-template-columns: 50px 50px 50px;
  grid-template-rows: 50px 50px;
}

This creates:

A 3x2 grid (3 columns, 2 rows)

Grid Template Shorthand

Instead of defining rows and columns separately:

/* Longhand */
.container {
  grid-template-columns: 50px 50px 50px;
  grid-template-rows: 50px 50px;
}

/* Shorthand: rows / columns */
.container {
  grid-template: 50px 50px / 50px 50px 50px;
}
Remember: Rows before the slash, columns after!

Lines and Tracks üìè

Grid Lines

  • Lines separate rows/columns
  • Numbered starting at 1
  • Created automatically
  • Used for positioning

Grid Tracks

  • Space between two lines
  • Can be rows or columns
  • Have a defined size
  • Where content lives
Pro Tip: Use DevTools to visualize grid lines! Look for the "grid" badge in your browser inspector.

Grid Cells üì¶

A grid cell is the space where a single row track and single column track intersect.

Think of it like a spreadsheet cell - defined by its row and column!

Positioning Items üìç

Use grid lines to position items across multiple cells:

.item {
  grid-column-start: 1;
  grid-column-end: 3;
  grid-row-start: 1;
  grid-row-end: 2;
}

This item spans from column line 1 to 3 (2 columns wide)

Remember: Grid lines, not cells! Line 1 to 3 = 2 cells.

Positioning Shorthand

/* Longhand */
.item {
  grid-column-start: 1;
  grid-column-end: 3;
  grid-row-start: 1;
  grid-row-end: 3;
}

/* Shorthand: start / end */
.item {
  grid-column: 1 / 3;
  grid-row: 1 / 3;
}

Grid Area (Two Ways!) üó∫Ô∏è

Method 1: Even Shorter Shorthand

/* row-start / col-start / row-end / col-end */
.item {
  grid-area: 1 / 1 / 3 / 3;
}

Method 2: Named Areas (Coming Up!)

.item {
  grid-area: header;
}

Example: Apartment Floor Plan üè†

Let's build a 5x5 grid for an apartment layout:

.apartment {
  display: inline-grid;
  grid-template-columns: repeat(5, 100px);
  grid-template-rows: repeat(5, 100px);
  gap: 10px;
}

#living-room {
  grid-column: 1 / 6; /* Full width */
  grid-row: 1 / 3; /* 2 rows tall */
}

Grid Template Areas üó∫Ô∏è

The most visual way to define layouts!

.container {
  display: grid;
  grid-template-areas:
    "header header header"
    "sidebar content content"
    "footer footer footer";
}

.header { grid-area: header; }
.sidebar { grid-area: sidebar; }
.content { grid-area: content; }
.footer { grid-area: footer; }

Empty Cells ‚ö™

Use a period (.) to indicate empty cells:

grid-template-areas:
  "header header header"
  "sidebar content ."
  "footer footer footer";

The top-right cell will be empty!

Explicit vs Implicit Grid ü§î

Explicit Grid

  • Tracks YOU define
  • Using grid-template properties
  • Fixed number of rows/columns

Implicit Grid

  • Tracks created AUTOMATICALLY
  • When you have more items
  • Browser adds rows/columns as needed
.container {
  grid-template-columns: 100px 100px; /* 2 columns */
  grid-template-rows: 100px 100px; /* 2 rows */
  grid-auto-rows: 100px; /* Size for auto rows */
}

The repeat() Function üîÑ

Instead of typing the same size over and over:

/* Tedious way */
grid-template-columns: 150px 150px 150px 150px 150px;

/* Better way */
grid-template-columns: repeat(5, 150px);

/* Works for rows too */
grid-template-rows: repeat(2, 150px);
Syntax: repeat(count, size)

Fractional Units (fr) üìè

The fr unit distributes remaining space proportionally!
/* All columns equal width */
grid-template-columns: repeat(3, 1fr);

/* First column 2x wider than others */
grid-template-columns: 2fr 1fr 1fr;

/* Mix with fixed sizes */
grid-template-columns: 200px 1fr 1fr;

The gap Property üåâ

Add space between grid items (gutters):

.container {
  display: grid;
  gap: 20px; /* All gaps */
}

/* OR separate row/column gaps */
.container {
  row-gap: 10px;
  column-gap: 20px;
  /* Shorthand: row column */
  gap: 10px 20px;
}

The minmax() Function üìê

Set minimum and maximum sizes for tracks:

/* Columns between 150px and 200px */
grid-template-columns: repeat(3, minmax(150px, 200px));

/* At least 200px, can grow to fill space */
grid-template-columns: repeat(3, minmax(200px, 1fr));
Great for responsive designs! Content won't shrink too small or grow too large.

auto-fit and auto-fill üîÆ

Create responsive grids that adjust automatically!

/* As many columns as will fit */
grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));

This creates as many 200px+ columns as will fit, then stretches them to fill!

auto-fit vs auto-fill: auto-fit collapses empty tracks, auto-fill keeps them. Subtle difference!

Using Grid and Flexbox Together ü§ù

Best Practice:

  • Use Grid for page layout (2D structure)
  • Use Flexbox for components (1D alignment)
/* Grid for overall layout */
.page {
  display: grid;
  grid-template-columns: 200px 1fr;
}

/* Flex for navigation items */
.nav {
  display: flex;
  gap: 20px;
}
