Understanding var üìù

The Original Variable Declaration

var greeting = "Hello World!";
console.log(greeting); // "Hello World!"

var greeting = "Hi there!"; // ‚úÖ Can redeclare
greeting = "Hey!"; // ‚úÖ Can reassign
console.log(greeting); // "Hey!"
Note: var is function-scoped, not block-scoped!

var and Function Scope üîç

function example() {
  var x = 10;
  if (true) {
    var x = 20; // Same variable!
    console.log(x); // 20
  }
  console.log(x); // 20 (not 10!)
}

console.log(x); // ReferenceError: x is not defined
‚ö†Ô∏è Warning: var ignores block scope‚Äîit only respects function boundaries!

var Hoisting ‚¨ÜÔ∏è

Variables are "lifted" to the top

What You Write

console.log(name);
var name = "Alice";
console.log(name);

How JavaScript Sees It

var name;
console.log(name); // undefined
name = "Alice";
console.log(name); // "Alice"

Problems with var ‚ö†Ô∏è

Why we needed something better

1. Loop Counter Confusion

for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 100);
}
// Prints: 3, 3, 3 (not 0, 1, 2!)

2. Accidental Global Variables

if (true) {
  var temp = 42;
}
console.log(temp); // 42 (leaked outside!)

Enter let üéâ

Block-Scoped Variables (ES6)

let message = "Hello!";
console.log(message); // "Hello!"

message = "Hi!"; // ‚úÖ Can reassign
let message = "Hey!"; // ‚ùå Cannot redeclare

if (true) {
  let blockScoped = "I'm local";
}
console.log(blockScoped); // ReferenceError

let and Block Scope üîí

Variables stay within their blocks

let x = 10;
console.log(x); // 10

if (true) {
  let x = 20; // Different variable!
  console.log(x); // 20
}

console.log(x); // 10 (unchanged!)
Block = Anything between curly braces { }

let Fixes Loop Issues üîÅ

Each iteration gets its own variable

for (let i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 100);
}
// Prints: 0, 1, 2 ‚úÖ

console.log(i); // ReferenceError: i is not defined

Why it works: Each loop iteration creates a new block scope with its own i

The Temporal Dead Zone (TDZ) ‚è∞

Variables exist but cannot be accessed

console.log(name); // ReferenceError!
let name = "Alice";
console.log(name); // "Alice"

TDZ: The period between entering a scope and the variable declaration. During this time, the variable exists but throws an error if accessed.

Best Practice: Always declare variables at the top of their scope!

Introducing const üîê

Block-Scoped Constants

const PI = 3.14159;
console.log(PI); // 3.14159

PI = 3.14; // ‚ùå TypeError!
const PI = 3; // ‚ùå SyntaxError!
Key Point: const creates a constant reference, not a constant value!

const Must Be Initialized ‚ö°

‚ùå Wrong

const name;
name = "Alice";
// SyntaxError!

‚úÖ Correct

const name = "Alice";
console.log(name);
// "Alice"
Rule: const variables must be assigned a value when declared!

const Block Scope üîí

Same scoping rules as let

const x = 10;
console.log(x); // 10

if (true) {
  const x = 20; // Different constant!
  console.log(x); // 20
}

console.log(x); // 10

console.log(y); // ReferenceError (TDZ)
const y = 5;

const: Primitive vs Reference üîç

Primitive Types

const num = 42;
num = 43; // ‚ùå Error

const str = "hi";
str = "bye"; // ‚ùå Error

Reference Types

const arr = [1, 2];
arr.push(3); // ‚úÖ OK
arr = []; // ‚ùå Error

const obj = {a: 1};
obj.b = 2; // ‚úÖ OK
obj = {}; // ‚ùå Error

const Rules üìã

‚ùå What You Can't Do

const greeting = "Hi";
greeting = "Hello"; // Error!

const greeting = "Hi";
const greeting = "Hey"; // Error!

const name; // Error!
name = "John";

‚úÖ What You Must Do

// Initialize at declaration
const greeting = "Hi";

// Use in different scopes
if (true) {
  const greeting = "Hello";
}

const with Objects & Arrays üé®

The reference is constant, not the contents!

Objects

const person = {
  name: "Alice",
  age: 30
};

// ‚úÖ Can modify properties
person.age = 31;
person.city = "NYC";

// ‚ùå Can't reassign
person = {}; // Error!

Arrays

const numbers = [1, 2, 3];

// ‚úÖ Can modify contents
numbers.push(4);
numbers[0] = 10;

// ‚ùå Can't reassign
numbers = [5, 6]; // Error!
Remember: const prevents reassignment of the variable itself, not modification of object/array contents!

Quick Comparison ‚öñÔ∏è

When to use each declaration

var

‚ùå Almost never! Only in legacy code or very specific cases

let

‚úÖ When you need to reassign the variable

  • Loop counters: for (let i = 0; ...)
  • Accumulator values
  • Conditional assignments

const

‚úÖ Default choice! Use unless you need to reassign

  • Configuration values
  • DOM element references
  • Imported modules
  • Most variables in modern JS

Practical Examples üíª

Real-world usage patterns

// Configuration values - use const
const API_URL = "https://api.example.com";
const MAX_RETRIES = 3;

// DOM elements - use const
const button = document.querySelector('.btn');
const container = document.getElementById('app');

// Values that change - use let
let currentUser = null;
let isLoading = false;

// Loops - use let for counters
for (let i = 0; i < 10; i++) {
  console.log(i);
}

Common Pitfalls üï≥Ô∏è

Mistakes to avoid

1. Using var in new code

Always prefer let or const in modern JavaScript

2. Overusing let when const would work

// ‚ùå Unnecessary let
let name = "Alice";
console.log(name);

// ‚úÖ Use const instead
const name = "Alice";
console.log(name);
Rule of Thumb: Use const by default, let only when reassignment is needed. Your IDE will tell you if you need let!

Best Practices Summary ‚ú®

The Modern JavaScript Way

‚úÖ DO:

  • Use const by default
  • Use let when you need to reassign
  • Declare variables at the top of their scope
  • Use descriptive variable names

‚ùå DON'T:

  • Use var in new code
  • Reassign const variables
  • Access variables before declaration (TDZ)
  • Create unnecessary global variables

Refactoring Exercise üîß

Convert var to let/const

Before (var)

  count++;
