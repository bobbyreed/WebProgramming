Historical Context üìú

CSS Saved HTML from Itself

The Dark Ages of Web Design

  • In the early web, complex designs were built with <table> elements
  • Separating HTML from visual styles became possible when CSS was widely adopted by browsers in the late '90s
  • CSS opened the door to completely changing a website's look without touching HTML

The CSS Revolution

This capability inspired projects like The CSS Zen Garden, created to demonstrate CSS's power and encourage developers to learn it.

Source: web.dev - "Layout"

CSS Foundations üé®

Cascading Style Sheets

/* CSS Rule Structure */
selector {
  property: value;
  another-property: value;
}

Three Ways to Add CSS:

  1. Inline: style="color: red;" (avoid!)
  2. Internal: <style> tags (sometimes ok)
  3. External: <link rel="stylesheet"> (best!)

CSS Selectors

Basic Selectors

/* Type */
p { }

/* Class */
.classname { }

/* ID */
#idname { }

Combinators

/* Descendant */
div p { }

/* Child */
div > p { }

/* Multiple */
h1, h2, h3 { }

CSS Specificity

Inline > ID > Class > Type

Specificity Values:

  • Inline styles: 1000 points
  • IDs: 100 points
  • Classes: 10 points
  • Elements: 1 point
Remember: Higher specificity wins! When in doubt, check DevTools.

The CSS Box Model

CONTENT
.box {
  width: 200px;
  padding: 20px;
  border: 5px solid black;
  margin: 10px;
}

Block vs Inline Elements

Block Elements

  • Full width by default
  • Start on new line
  • Can set width/height
  • Examples: div, p, h1-h6

Inline Elements

  • Only needed width
  • Stay in line
  • Can't set width/height
  • Examples: span, a, strong
Inline-block: Best of both worlds!

Practice: CSS Cascade

<!-- HTML -->
<div class="container" id="main">
  <p class="text">What color am I?</p>
</div>

/* CSS */
p { color: blue; }
.text { color: green; }
#main p { color: red; }
div p { color: yellow; }

Question: What color will the text be?

Answer: Red - ID selector has higher specificity!

Historical Context ‚ö†Ô∏è

HTML Told Us NOT to Use Tables for Layout in 1997

The Evolution of Web Standards

  • 1997: Tables existed in HTML standard with a warning
  • April 1998 (HTML 4): Warning strengthened: "Tables should not be used purely as a means to layout document content"
  • December 1999 (HTML 4.01): Became a directive, not just a warning
  • Early 2000s: Days spent trying to get layouts to work properly with CSS1

Implementation of CSS1 was slow, but the standards were clear from the beginning!

Source: Tiernok.com - "The History of HTML Table Layouts"

Flexbox Fundamentals üì¶

/* Make a flex container */
.container {
  display: flex;
}

Container becomes flexible!

  • Children become flex items
  • Items flow in a row by default
  • Items can grow, shrink, and be aligned

Flex Container Properties

.container {
  display: flex;
  flex-direction: row; /* row | column */
  justify-content: center; /* main axis */
  align-items: center; /* cross axis */
  flex-wrap: wrap; /* wrap | nowrap */
  gap: 20px; /* space between items */
}
Remember: Main axis changes with flex-direction!

Flex Item Properties

.item {
  flex-grow: 1; /* growth factor */
  flex-shrink: 1; /* shrink factor */
  flex-basis: auto; /* initial size */
  
  /* Shorthand */
  flex: 1; /* grow: 1, shrink: 1, basis: 0 */
}
Common Pattern: flex: 1 makes items share space equally

Common Flexbox Patterns

Center Everything

.center {
  display: flex;
  justify-content: center;
  align-items: center;
}

Navigation Bar

.nav {
  display: flex;
  justify-content: space-between;
  align-items: center;
}

Historical Context üöÄ

From Tables to Flexbox: A 20-Year Journey

The Evolution of Web Layout

  • Late 1990s: Major movement to bring order to HTML‚ÄîNetscape and Microsoft agreed to phase out presentational tags
  • 2000s: CSS allowed style information to be kept separate from structure
  • 2011: Bootstrap 3 was written using floats for layout
  • 2018: Bootstrap 4 adopted Flexbox
  • Today: CSS Grid is the first true two-dimensional layout system

CSS Grid solves the layout challenges of the past. No more floats, hacks, or even frameworks needed!

Source: Grid-Layout.com - "Web Layout History"
